## Prose math

Prose math renders through the `remark-math` / `rehype-katex` pipeline already wired
into `vite.config.ts` — no explicit component needed. For example, the softmax policy
used in the paper is $\pi(a \mid s) \propto \exp(Q(s, a) / \tau)$, where $\tau$ is an
inverse-temperature parameter controlling exploration.

## Component math

The `<Tex>` primitive renders the same expressions from a variable or computed string,
for cases where the expression isn't static prose:

<Tex expr="\pi(a \mid s) \propto \exp(Q(s, a) / \tau)" block />

Both paths should render identically — the difference is only where the expression
comes from (static prose vs. a component prop).

## Runnable Python

The `<PyRepl>` primitive runs real Python in the browser via
[Pyodide](https://pyodide.org) — no backend, no build step. Edit the code and press
Run:

<PyRepl code={"import sys\nprint(sys.version)\nfor i in range(5):\n    print(i ** 2)"} />
